# 案件入力・請求管理システム 運用マニュアル

このドキュメントは、LINEを使った案件入力・請求管理システムを運用し、組織内に定着させるための操作手順と仕組みをまとめたものです。

本システムは以下の4つのサービスが連携して動いています。
1. **GitHub**: 入力フォーム（画面）の管理と公開
2. **Google Apps Script (GAS)**: データの保存・裏側の処理
3. **LINE Developers**: LINEアプリと入力フォームの連携設定
4. **LINE公式アカウント**: スタッフ向けの窓口とメニュー表示

それぞれの役割と操作方法を以下に詳しく解説します。

---

## 1. GitHub の操作方法（フロントエンド画面の更新）

**【役割】**
スマホで開く「案件入力フォーム」の見た目（HTML/CSS/JavaScript）の管理とWeb上への公開（GitHub Pages）を行います。

**【操作手順（画面デザインや項目を変更したい場合）】**
1. ローカルパソコンにある `liff-frontend` フォルダ内の `index.html` などのファイルを修正します。
2. ターミナル（コマンドプロンプトやPowerShell）で `liff-frontend` フォルダを開き、以下のコマンドを実行して変更を送信（プッシュ）します。
   ```bash
   git add index.html
   git commit -m "変更内容のメモをここに書く"
   git push origin main
   ```
3. プッシュ後、1〜2分待つと **GitHub Actions** によって自動的にページが更新されます。
4. ※変更が反映されない場合は、スマホやパソコンのブラウザに古いデータ（キャッシュ）が残っている可能性が高いため、スーパーリロード（再読み込み）を行ってください。

---

## 2. Google Apps Script (GAS) の操作方法（バックエンド・データの更新）

**【役割】**
フロントエンドから送信されたデータを受け取り、Googleスプレッドシートに書き込む処理を行います。

**【操作手順（処理ロジックの変更や新しい機能を追加したい場合）】**
1. ローカルの `gas` フォルダ内のファイル（`Code.js` など）を編集します。
2. 編集が終わったら、ターミナルで `clasp push` を実行し、GASのエディタ（クラウド上）に変更を反映させます。
3. **【重要】GASのURLを更新する手順**:
   - スクリプトの内容を大きく変更した場合、GASのエディタ画面右上の「デプロイ」>「新しいデプロイ」または「デプロイを管理」から、**新バージョンのWebアプリ**として再公開する必要があります。
   - デプロイ後に発行される「ウェブアプリのURL」をコピーします。
   - `liff-frontend/index.html` の `const GAS_URL = '...'` の部分を新しいURLに書き換え、GitHubへプッシュ（手順1を参照）してください。

**💡 運用のコツ**:
案件名、単価、スタッフ名などのデータはプログラムをいじる必要はありません。**スプレッドシートのマスターデータ**を直接書き換えるだけで、入力フォームの選択肢に自動で反映されます。

---

## 3. LINE Developers の操作方法（LIFFアプリの設定）

**【役割】**
LINEアプリのトーク画面からWebページ（入力フォーム）を綺麗に開くための橋渡し（LIFF機能）を提供します。

**【操作手順（LIFF URLの取得や接続先を変更したい場合）】**
1. [LINE Developersコンソール](https://developers.line.biz/) にログインします。
2. 作成したプロバイダーを選択し、対象の「LINEログインチャネル」を開きます。
3. 「LIFF」タブを開き、LIFFアプリの一覧を確認します。
4. **エンドポイントURL**: ここに GitHub Pages の公開URL（`https://minagawaharuto.github.io/LINE_frontend/`）を設定します。
5. **LIFF ID**: 登録すると `1234567890-abcdefg` のようなIDが発行されます。これを `index.html` 内の `LIFF_ID` に設定します。
6. 設定が完了すると、LIFF URL（`https://liff.line.me/○○○○`）が発行されます。**このURLをLINE公式アカウントのリッチメニューに設定します。**

---

## 4. LINE公式アカウント (Official Account Manager) の操作方法

**【役割】**
スタッフが日常的に利用する窓口です。リッチメニュー（画面下部のボタン）を設置し、そこから入力フォームを開けるようにします。

**【操作手順（リッチメニューを設定・変更したい場合）】**
1. [LINE Official Account Manager](https://manager.line.biz/) にログインします。
2. 左側メニューの「トークルーム管理」＞「リッチメニュー」を開き、「作成」をクリックします。
3. **背景画像**: 作成したリッチメニュー用の画像（1200×810pxなど）をアップロードします。
4. **アクション設定**: 
   - タイプを「リンク」に設定します。
   - **URL**: LINE Developersで取得した **LIFF URL**（`https://liff.line.me/○○○○`）を入力します。
   - ※注意：GitHubのURL（https://minagawaharuto.github.io/...）を直接入力すると、LINEのプロフィール情報などがうまく連携されない場合があるため、必ずLIFF URLを指定してください。
5. 表示期間を設定し、「保存」を押せば設定完了です。

## システムを組織に定着させるためのヒント (Tips for Adoption)

1. **アクセス手順の簡略化**
   - 「LINEを開いて下のボタンを押すだけ」というシンプルな手順を徹底し、マニュアル化してスタッフに共有してください。
   - 新規スタッフが入った際は、まず公式アカウントを友だち追加してもらうだけで準備が完了します。
2. **マスターデータの定期メンテナンス**
   - 選択肢が多すぎると入力ミスや手間の原因になります。スプレッドシートのマスターデータを月に1回など定期的に見直し、不要な店舗や案件名は削除（または非表示）にしましょう。
3. **エラー発生時の一次対応フロー**
   - 「ページが開かない」「エラーが出る」という問い合わせがあった場合は、まずは**「一度タスクキル（LINEアプリを再起動）する」**、または**「フォームのURLを読み込み直す」**ことを案内してください。多くの場合キャッシュの問題で解決します。
   - それでも解決しない場合は、GASのURLやGitHubのデプロイ状況を確認するエスカレーションフローを決めておくとスムーズです。
