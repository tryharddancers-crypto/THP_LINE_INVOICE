## 1. システム概要

### 1.1 目的

ダンサー等の外注スタッフが、LINE公式アカウントを通じて日々の案件（業務）情報をスマートフォンから簡単に登録できる仕組みを提供する。  
登録されたデータは自動的に Google スプレッドシート（月次外注連絡表）に集計・記録され、月末の明細PDF生成・メール送信まで一貫して自動化する。

### 1.2 解決する課題

- 紙や個別チャットでの業務報告による転記ミス・漏れ
- 月次集計・明細書作成に要していた手作業の負担
- スタッフへの個別メール送信の手間

### 1.3 システム名

**LINE請求書自動作成システム**（内部コード: LINE_INVOICE）

```

                             │ HTTPS POST / GET
┌────────────────────────────▼────────────────────────────────────┐
│              Google Apps Script（GAS）Webアプリ                  │
│  ・フォームHTMLの配信（doGet）                                   │
│  ・案件データの受信・処理（doPost）                              │

│  ・PDF生成・メール送信                                           │
│  ・月次シート自動作成（時間トリガー）                            │
└──────┬──────────────────────────────────┬───────────────────────┘
       │                                  │
┌──────▼──────┐                  ┌────────▼────────┐
│マスタ        │                  │月次スプレッドシート│
```

## 3. ユーザー・登場人物

| ロール | 説明 | 主な操作 |
|--------|------|---------|
| **スタッフ（ダンサー等）** | 日々の案件情報を入力するエンドユーザー | LIFFフォームから案件を登録する |
| **管理者** | スプレッドシートとGASを管理する担当者 | マスタ管理、PDF一括送信、データ修正 |
| **システム（GAS）** | 自動実行されるバックグラウンド処理 | 月次シート生成、PDF自動送信 |

## 4. 機能要件

### 4.1 フロントエンド（LIFFアプリ）

#### FR-01：マスタデータ取得・表示

- システム起動時、GAS APIエンドポイント（`?action=getMaster`）から最新の案件一覧・ダンサー名一覧を取得する
- 店舗名（請求元）ドロップダウンを動的に生成する
- 店舗選択後、対応する案件（商品）名のみを案件ドロップダウンに絞り込み表示する
- ダンサー名一覧をプルダウンに表示する

#### FR-02：案件入力フォーム

| 入力項目 | 説明 | 必須 |
|---------|------|------|
| 店舗名 | 請求元店舗（案件マスタの `billing` 列） | ○ |
| 日程 | 業務実施日（デフォルト: 本日） | ○ |
| 名前 | ダンサー名（ダンサーマスタから選択） | ○ |
| 商品名 | 案件名（店舗選択後に絞り込み） | ○ |
| 数量 | 1以上の整数 | ○ |
| 詳細 | 任意のメモ・特記事項 | － |

- 1つのフォームに複数人物（人物カード）を追加可能
- 1人物につき複数案件（商品）を追加可能
- 人物カードおよび案件アイテムの個別削除が可能

#### FR-03：送信前確認画面

- 入力内容を人物・日付ごとにグループ化して表示する
- 「修正する」ボタンで入力フォームへ戻ることができる
- 「確定送信」ボタンで最終送信を実行する
- バリデーションエラーは明細フォーム画面に赤文字で表示する

#### FR-04：送信処理

- GAS Webアプリ（doPost）に JSON ペイロード（`userId`, `rows`）を POST する
- 送信中はボタンを非活性化し「送信中...」を表示する
- 成功時は件数と日付を表示し、2秒後にフォームをリセット・初期化する
- 失敗時はエラーメッセージを表示し、ボタンを再活性化する

#### FR-05：LINE ログイン

- LIFF SDK を使用し、未ログインの場合は自動的に LINE ログインページへリダイレクトする
- ログイン後にユーザープロフィール（`userId`）を取得し、送信データに含める

---

### 4.2 バックエンド（Google Apps Script）

#### FR-06：マスタデータ提供 API

- `GET ?action=getMaster` に対して以下のデータをJSON形式で返す
  - `jobList`: `[{ name, billing, unitPrice }]`
  - `dancerNames`: `[string]`
- 「判定表」シートが存在する場合、ダンサーマスタと判定表の両方に存在する名前のみを返す

#### FR-07：案件データ受信・スプレッドシート書き込み

- `POST` リクエストを受信し、`rows` 配列からデータを抽出する
- 入力日付の年月に対応する月次スプレッドシートを取得または新規作成する
- 「2.入力表」シートの既存データ末尾に行を追記する
  - 書き込み列: C=日程、D=曜日、G=案件名、J=詳細、K=名前、L=数量
  - B（チーム）・E（現場）・F（項目）・M（単価）・N（合計）はシート内の数式で自動計算
- データ書き込み後、`SpreadsheetApp.flush()` で確実に反映する

#### FR-08：LINE プッシュ通知

- データ書き込み完了後、送信者の LINE ユーザーID に対してプッシュメッセージを送信する
- メッセージ内容: 登録件数、人物名、案件名・数量の一覧、再入力用 LIFF URL
- LINE Messaging API `push` エンドポイントを使用する

#### FR-09：PDF 自動生成・メール送信（フォーム送信時）

- フォーム送信後、登録されたデータに含まれる人物ごとに自動的に実行する
- ダンサーマスタの L 列（12列目）からメールアドレスを取得する
- 「明細書テンプレート」シートに対象人物・日付のデータを書き込み PDF 化する
- 生成した PDF を GmailApp で添付送信する（差出人名: 管理事務局）
- メールアドレス未登録の場合は送信をスキップし、エラーログを出力する

#### FR-10：PDF 一括送信（管理者手動実行）

- スプレッドシートのメニュー「PDF送信 > 明細PDFを一括送信」から実行する
- 「2.入力表」の全データを人物ごとに集計する
- 全対象人物に対して FR-09 と同様の PDF 生成・メール送信を行う
- 処理完了後、送信件数・エラー内容をダイアログで表示する

#### FR-11：月次スプレッドシート自動作成

- 毎月 1 日 0時台に時間トリガーで実行する（`createMonthlySheetTrigger`）
- テンプレートスプレッドシートを対象年月名でコピーし、年フォルダに保存する
- ファイル名形式: `【ダンサー】令和〇年〇月分 外注連絡表`
- 作成時、「外注連絡票」シートにタイトル（和暦）とダンサーマスタの情報を転記する

#### FR-12：マスタ初期セットアップ

- Excelファイル（`.xlsx`）から「外注連絡票」「判定表」「金額」の各シートを読み込み、マスタスプレッドシートへ転記する（`setupMasterFromExcel`）
- ダンサーマスタへの転記: 芸名、本名、コード、時給、交通費、在籍、金融機関情報、住所、連絡先
- 案件マスタへの転記: 案件名、現場名（現場コード）、単価、項目区分

## 5. 非機能要件

### 5.1 可用性

- GAS Webアプリは Google のインフラに依存するため、独自の冗長化は不要
- Google サービスの障害時は業務停止を許容する

### 5.2 パフォーマンス

- フォーム送信後のレスポンスは原則 **10 秒以内** とする（PDF生成を含む処理全体）
- マスタデータ取得（`getMaster`）は **3 秒以内** とする

### 5.3 スケーラビリティ

- ダンサー数: 最大 **100 名程度**を想定
- 月次データ行数: 最大 **500 行/月程度**を想定
- 同時利用者数: **5 名以下**を想定（GAS の同時実行制限の範囲内）

### 5.4 ユーザビリティ

- スマートフォン（iOS / Android）での操作を主対象とする
- LINE アプリ内ブラウザ（LIFF）で正常動作すること
- 専用 ID・パスワード不要（LINE アカウントで認証）

### 5.5 保守性

- GAS コードは clasp で Git 管理する
- 関数・変数は日本語コメント付きで記述し、保守担当者が理解できる状態を維持する

## 6. データ設計

### 6.1 マスタスプレッドシート

#### 案件マスタシート

| 列 | 項目 | 型 | 説明 |
|----|------|-----|------|
| A | 案件名 | 文字列 | LIFFフォームの商品ドロップダウンに表示 |
| B | 現場コード（請求元） | 文字列 | 店舗名としてドロップダウンに表示 |
| C | 単価 | 数値 | 1件あたりの金額 |
| D | 項目区分 | 文字列 | 費目の分類 |

#### ダンサーマスタシート

| 列 | 項目 | 型 | 説明 |
|----|------|-----|------|
| A | 芸名 | 文字列 | LIFFフォームの名前ドロップダウンに表示 |
| B | 本名 | 文字列 | 外注連絡票に記載 |
| C | コード | 文字列 | 管理コード |
| D | 時給 | 数値 | |
| E | 交通費 | 数値 | |
| F | 在籍 | 文字列 | 在籍ステータス |
| G | 金融機関 | 文字列 | 振込先銀行名 |
| H | 口座番号 | 文字列 | |
| I | 口座名義(カナ) | 文字列 | |
| J | 住所 | 文字列 | |
| K | 連絡先 | 文字列 | |
| L | メールアドレス | 文字列 | 明細PDF送信先 |

---

### 6.2 月次スプレッドシート

#### 2.入力表シート（データ開始行: 8行目）

| 列 | 項目 | 書き込み元 |
|----|------|-----------|
| B | チーム（判定） | 自動計算（判定表参照） |
| C | 日程 | GAS書き込み |
| D | 曜日 | GAS書き込み |
| E | 現場 | 自動計算（金額シート参照） |
| F | 項目 | 自動計算（金額シート参照） |
| G | 案件名 | GAS書き込み |
| J | 詳細 | GAS書き込み |
| K | 名前 | GAS書き込み |
| L | 数量 | GAS書き込み |
| M | 単価 | 自動計算 |
| N | 合計金額 | 自動計算 |

#### 外注連絡票シート

- ダンサーマスタの情報を4行目以降に転記（在籍、コード、芸名、氏名、金融機関情報）
- タイトル: `≪令和〇年〇月分 外注費支払表≫`（A1セル）

## 7. 外部連携

### 7.1 LINE Messaging API

| 項目 | 内容 |
|------|------|
| エンドポイント | `https://api.line.me/v2/bot/message/push` |
| 認証方式 | チャネルアクセストークン（Bearer Token） |
| メッセージ種別 | テキストメッセージ |
| 送信タイミング | フォーム送信完了後（GAS doPost 内） |

### 7.2 LIFF SDK

| 項目 | 内容 |
|------|------|
| SDK バージョン | LIFF SDK v2 |
| 読み込み元 | `https://static.line-scdn.net/liff/edge/2/sdk.js` |
| 使用機能 | `liff.init()`, `liff.login()`, `liff.isLoggedIn()`, `liff.getProfile()` |

### 7.3 Google Drive API（GAS 内部）

- `DriveApp.getFolderById()` で月次保存フォルダを取得
- 年別サブフォルダの自動作成
- テンプレートファイルのコピーによる月次ファイル生成

### 7.4 Gmail（GAS 内部）

- `GmailApp.sendEmail()` で明細PDF添付メールを送信
- 差出人名: `管理事務局`

## 8. セキュリティ要件

| 要件 | 内容 |
|------|------|
| 認証 | LINE アカウントによる LIFF 認証（`liff.login()`） |
| GAS Webアプリ公開設定 | 「全員」に公開（LIFF からアクセスするため） |
| 機密情報管理 | `LINE_CHANNEL_ACCESS_TOKEN` 等は GAS スクリプトプロパティで管理（コードに直接記述しない） |
| 入力バリデーション | フロントエンドでの必須チェック、バックエンドでの `rows` 空チェック |
| PDF送信先制限 | マスタスプレッドシートに登録されたメールアドレスのみに送信 |

> **注意**: GAS Webアプリは全員公開のため、URL が漏洩した場合に不正アクセスのリスクがある。機密データを含む操作は LIFF の `userId` チェックを追加することが推奨される。

## 9. 運用・保守要件

### 9.1 スクリプトプロパティ（設定値）

| キー | 説明 |
|------|------|
| `MASTER_SPREADSHEET_ID` | マスタスプレッドシートの ID |
| `TEMPLATE_SPREADSHEET_ID` | 月次テンプレートスプレッドシートの ID |
| `MONTHLY_FOLDER_ID` | 月次ファイルの保存先 Google Drive フォルダ ID |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Messaging API チャネルアクセストークン |
| `LIFF_ID` | LIFF アプリ ID |
| `DANCER_EXCEL_SPREADSHEET_ID` | 初回セットアップ用 Excel スプレッドシート ID（初回のみ） |

### 9.2 定期実行トリガー

| 関数名 | 実行タイミング | 説明 |
|--------|--------------|------|
| `createMonthlySheetTrigger` | 毎月1日 0:00〜1:00 | 当月の月次スプレッドシートを自動作成 |

### 9.3 マスタデータの更新方法

- **ダンサーマスタへの追加**: マスタスプレッドシート「ダンサーマスタ」シートの末尾に行を追加
- **案件マスタへの追加**: マスタスプレッドシート「案件マスタ」シートの末尾に行を追加
- スプレッドシート保存後、LIFF フォームを開き直すと自動的に最新データが反映される

### 9.4 コードのデプロイ方法

```bash
# GASへアップロード
clasp push

# GASからダウンロード
clasp pull
```

- GAS エディタで「デプロイ > 新しいデプロイ」→「ウェブアプリ」として公開する
- デプロイ時の設定: 実行ユーザー=「自分」、アクセス=「全員」

## 10. 制約・前提条件

### 10.1 外部サービス依存

- Google アカウント（オーナー）が有効であること
- Google スプレッドシート、Google Drive、Gmail の利用制限に収まること
- LINE 公式アカウントおよび LIFF チャネルが有効であること
- LINE チャネルアクセストークンの有効期限に注意すること（長期トークン推奨）

### 10.2 GAS 実行制限（Google の制約）

| 制限 | 上限 |
|------|------|
| スクリプト実行時間 | 6 分 / 実行 |
| 1 日のトリガー実行回数 | 20 回 / ユーザー |
| メール送信数 | 100 通 / 日（無料アカウント） |

### 10.3 データ修正について

- 一度送信されたデータの修正は LIFF フォームからはできない
- スプレッドシートを直接編集するか、管理者に依頼する

### 10.4 その他前提

- スタッフは LINE アカウントを保有していること
- スタッフはスマートフォン（iOS または Android）から操作すること
- 管理者は Google アカウントへのアクセス権を持つこと
