## 1. システム概要

### 目的

ダンサー（外注スタッフ）が毎月の稼働実績をLINEアプリ内のフォームから入力し、管理側が自動的に請求明細書を生成・配布するシステム。

### 解決する課題

| 従来の課題 | システムによる解決 |
|-----------|----------------|
| 紙や口頭での稼働報告 | LINEアプリ内フォームで簡単入力 |
| 単価の手動調べ | マスタから自動取得・リアルタイム表示 |
| 明細書の手作り | PDF自動生成・自動メール送信 |
| 報告した確認が取れない | LINE通知で即時フィードバック |
| 月次ファイル管理が手間 | Google Driveに自動フォルダ・ファイル作成 |

### 主な機能

1. **稼働入力フォーム** - LINE内で複数人・複数案件を一括入力
2. **リアルタイム金額確認** - 単価×数量を入力中に表示
3. **LINE通知** - 送信後に確認メッセージをプッシュ
4. **PDF自動生成** - 送信のたびに明細書PDFをメール送信
5. **月次スプレッドシート自動作成** - 毎月1日に自動でファイル作成
6. **一括PDF送信** - 月末にすべてのダンサーへ明細書を一括メール

## 5. コンポーネント詳細

### 5.1 フロントエンド（LIFFアプリ）

**ファイル**: `liff-frontend/index.html`

#### 画面構成

```
┌─────────────────────────────────┐
│  請求情報入力フォーム             │
│                                  │
│  [請求元/カテゴリ ▼]             │← グローバル選択（全行に適用）
│                                  │
│  ┌─────────────────────────────┐ │
│  │ 日程: [日付入力]             │ │
│  │ 名前: [ダンサー ▼]           │ │
│  │                             │ │
│  │ 案件1: [案件名 ▼] [数量]    │ │
│  │        ¥単価 × 数量 = 小計  │ │
│  │ [詳細 (任意)]               │ │
│  │ [+ 案件追加]                 │ │
│  └─────────────────────────────┘ │
│  [+ 人物追加]                    │
│                                  │
│  [内容を確認する]                 │
└─────────────────────────────────┘
```

#### 処理フロー

```javascript
// 1. 初期化
liff.init({ liffId })
→ liff.getProfile()  // LINE ユーザーID取得
→ GET GAS_URL?action=getMaster  // 案件・ダンサー一覧取得

// 2. フォーム操作
カテゴリ選択 → 案件ドロップダウン絞り込み
案件選択 → 単価を priceMap から取得してリアルタイム表示

// 3. 確認画面
collectRows()  // バリデーション + データ収集
→ 人物×日付でグループ化して表示
→ 小計・合計を計算して表示

// 4. 送信
POST GAS_URL { userId, rows: [...] }
→ 成功: "✅ N件を送信しました" 表示 → フォームリセット
→ 失敗: エラーメッセージ表示（再送信可能）
```

### 5.2 バックエンド（Google Apps Script）

#### Code.js - メインエントリポイント

```
doGet(e)
├── action=getMaster → getMasterData() をJSONで返す
└── それ以外 → LIFF HTMLテンプレートを返す（LIFF_ID, gasUrl 注入）

doPost(e)
└── handleSubmission(e)
    ├── rows バリデーション
    ├── getOrCreateMonthlySpreadsheet(date)  // 月次SSを取得/作成
    ├── appendRowsToInputSheet(ss, rows)     // 2.入力表に書き込み
    ├── buildSubmissionMessage(rows, liffUrl)// LINE メッセージ組み立て
    ├── sendLineMessage(userId, message)     // LINE プッシュ送信
    └── sendPdfForPerson(ss, name, date)    // PDF生成・メール送信（各人）
```

**LINE通知メッセージ形式**:
```
✅ 2件を追加しました

📅 2026/04/15
  ダンサーA: ○○案件 ×1, △△案件 ×2

LIFF URL: https://liff.line.me/...
```

#### MonthlySheet.js - 月次スプレッドシート管理

```
getOrCreateMonthlySpreadsheet(date)
├── Google Drive フォルダ構造を確認/作成
│   MONTHLY_FOLDER_ID/
│   └── {YYYY}年/
│       └── 【ダンサー】令和N年M月分 外注連絡表
│
├── 既存ファイルがあれば開いて返す
└── なければ TEMPLATE_SPREADSHEET_ID をコピーして作成
    ├── 外注連絡票の月ヘッダーを更新
    └── fillDancerMasterToSheet() でダンサー情報を事前記入

appendRowsToInputSheet(ss, rows)
→ 2.入力表 シートに各行を追記
   書き込み列:
   C: 日程 (YYYY/MM/DD)
   D: 曜日 (月〜日)
   G: 案件名
   J: 詳細（任意）
   K: 名前
   L: 数量
   ※ M列(単価)・N列(合計)はシート数式で自動計算

createMonthlySheetTrigger()
→ 毎月1日 00:00 に自動実行（時間主導型トリガー）
```

#### PdfSender.js - PDF生成・メール送信

```
sendPdfsToAll()  ← スプレッドシートメニューから手動実行
├── ダンサーマスタからメールアドレス一覧取得
├── 2.入力表 のデータをダンサー別にグループ化
└── 各ダンサーに対して:
    ├── 明細書テンプレート シートにデータ書き込み
```

- 添付: `{ダンサー芸名}_令和N年M月分_明細書.pdf`

### 5.3 データストア（Googleスプレッドシート）

システムで使用するスプレッドシートは3種類。

#### マスタスプレッドシート（常時1ファイル）

スクリプトプロパティ `MASTER_SPREADSHEET_ID` で参照。

| シート名 | 用途 |
|---------|------|
| 案件マスタ | 案件名・現場・単価・カテゴリの一覧 |
| ダンサーマスタ | ダンサーの個人情報・銀行情報一覧 |
| 判定表 | 有効なダンサー名の一覧（フィルタ用）|

#### テンプレートスプレッドシート（常時1ファイル）

スクリプトプロパティ `TEMPLATE_SPREADSHEET_ID` で参照。月次ファイル作成時にコピーされる。

| シート名 | 用途 |
|---------|------|
| 2.入力表 | 稼働実績の入力・集計 |
| 外注連絡票 | 支払い計算（数式内蔵）|
| 明細書テンプレート | PDF出力用の明細フォーマット |

#### 月次スプレッドシート（月ごとに1ファイル自動作成）

テンプレートをコピーして作成。Google Drive の以下の構造で保存：

```
MONTHLY_FOLDER_ID/
└── 2026年/
    ├── 【ダンサー】令和8年4月分 外注連絡表
    ├── 【ダンサー】令和8年5月分 外注連絡表
    └── ...
```

## 6. データフロー

### 6.1 フォーム送信フロー（日常業務）

```
1. ダンサーが LINE 公式アカウントのリッチメニューをタップ

2. LIFF アプリが起動
   ├── liff.init() でLINEログイン確認
   ├── liff.getProfile() で LINE ユーザーID取得
   └── GET GAS?action=getMaster でドロップダウン項目取得
       └── レスポンス: { jobList: [...], dancerNames: [...] }

3. ダンサーがフォーム入力
   ├── カテゴリ（請求元）を選択
   ├── 人物カードに日程・名前を入力
   ├── 案件・数量・詳細を入力
   └── 単価 × 数量 がリアルタイム表示される

4. 「内容を確認する」をタップ
   └── collectRows() でバリデーション & データ収集
       └── 人物+日付でグループ化した確認画面を表示

5. 「確定送信」をタップ
   └── POST GAS_URL
       Body: { userId: "U...", rows: [{ date, jobName, name, qty, detail, unitPrice }] }

6. GAS バックエンド処理
   ├── handleSubmission() が起動
   ├── 送信日から月を特定
   ├── getOrCreateMonthlySpreadsheet() で月次SSを取得/作成
   ├── appendRowsToInputSheet() で 2.入力表 に書き込み
   ├── buildSubmissionMessage() で通知メッセージ組み立て
   ├── sendLineMessage() で LINE プッシュ通知を送信
   └── sendPdfForPerson() で各ダンサーにPDFメール送信

7. ダンサーへのフィードバック
   ├── LIFFフォーム: "✅ N件を送信しました" 表示
   ├── LINE通知: 送信内容の確認メッセージ
   └── メール: 明細書PDF（送信した案件の一覧）
```

### 6.2 月次集計フロー

```
月末（または月初）

1. 毎月1日 00:00 に createMonthlySheetTrigger() が自動実行
   └── 翌月分の月次スプレッドシートを事前作成

2. 管理者が当月の月次スプレッドシートを確認
   └── Google Drive → MONTHLY_FOLDER_ID → {YYYY}年 → 対象月ファイル

3. 2.入力表 シートでデータを確認・修正

4. 外注連絡票 シートで支払い計算を確認
   └── 数式により自動計算:
       ├── 出演料 + 交通費
       ├── 源泉徴収税 (10.21%)
       ├── 差引支給額
       └── 立替精算後の振込額

5. スプレッドシートメニューの「PDF送信」→「明細PDFを一括送信」
   └── sendPdfsToAll() で全ダンサーに明細書PDFをメール送信

6. 経理部門が 外注連絡票 を参照して銀行振込処理
```

### 6.3 初期セットアップフロー

```
（システム導入時のみ）

1. 設定に必要なスプレッドシートとフォルダを準備
   ├── マスタスプレッドシートを作成（案件マスタ・ダンサーマスタシート）
   ├── テンプレートスプレッドシートを作成（3シート）
   └── 月次ファイル保管用Driveフォルダを作成

2. GAS スクリプトプロパティに各IDを設定

3. 既存のExcelデータをGoogleスプレッドシートに変換
   └── DANCER_EXCEL_SPREADSHEET_ID をスクリプトプロパティに設定

4. GAS エディタで setupMasterFromExcel() を実行
   └── Excelデータをマスタスプレッドシートにインポート

5. LINE Developers でチャンネル作成・設定
   ├── Messaging API チャンネルアクセストークン取得
   └── LIFF アプリを作成して LIFF_ID を取得

6. GAS スクリプトプロパティに LINE 関連情報を設定

7. GAS Webアプリとして公開（clasp push + ウェブアプリデプロイ）

8. LINE公式アカウントのリッチメニューにLIFF URLを設定
```

## 12. 運用フロー

### 日常業務（ダンサー側）

```
1. LINEを開く
2. 公式アカウントのリッチメニューをタップ
3. フォームに入力（カテゴリ・日程・案件・数量）
4. 確認画面で内容・金額を確認
5. 送信 → LINE通知 + メールで確認
```

### 日常業務（管理者側）

```
- 特別な操作は不要
- ダンサーの送信状況は月次SSの 2.入力表 で確認可能
- 問題のある入力があれば 2.入力表 を直接修正
```

### 月次業務（管理者側）

```
1. 月次SS を開く（Google Drive → MONTHLY_FOLDER_ID → {年} → 対象月）
2. 2.入力表 で全入力を確認・修正
3. 外注連絡票 で各ダンサーの支払い金額を確認
4. スプレッドシートメニューの「PDF送信」→「明細PDFを一括送信」を実行
5. 送信完了のアラートで件数とエラーを確認
6. 経理担当者へ外注連絡票を共有 → 振込処理
```

### マスタデータ更新（随時）

```
- 新案件追加: マスタSSの 案件マスタ シートに直接行を追加
- 新ダンサー追加: マスタSSの ダンサーマスタ シートに直接行を追加
- 変更は即時反映（フォーム起動時に毎回マスタを参照するため）
```

## 13. エラーハンドリング

| 箇所 | エラー種別 | 対応 |
|-----|----------|------|
| フォームバリデーション | 必須項目未入力 | エラーメッセージ表示（送信ブロック）|
| GAS doPost | rows未定義・空 | `{ok: false, error: "rows必須"}` を返す |
| LINE通知送信失敗 | HTTP 200以外 | 例外をスローしログに記録 |
| PDF生成失敗 | スプレッドシート操作エラー | ログに記録（フォーム処理はブロックしない）|
| PDF送信失敗 | Gmail API エラー | エラー詳細をログに記録・一括送信では結果サマリーに含める |
| マスタ参照失敗 | スプレッドシート不在 | 例外をスローしGAS ログに記録 |

### ログの確認方法

GASエディタ → 実行数（左メニュー）→ 各実行のログを確認
